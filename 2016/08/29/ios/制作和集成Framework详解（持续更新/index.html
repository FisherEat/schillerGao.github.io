集成framework报错

  1. error: module ‘Fabric’ was built in directory ‘/Fabric.framework’ but now resides in directory ‘./Fabric.framework’。

    这个报错是因为我更换了framework的路径，但是还是使用之前的framework，在.framework目录下有一个 Modules文件夹。

    解决方案：我是用的是脚本生成framework，删除生成framework的DriveData ,重新生成framework，重新导入framework，更改path为正确的路径。

    一般的，如果你更换了framework，那么path的路径会自动更换。
