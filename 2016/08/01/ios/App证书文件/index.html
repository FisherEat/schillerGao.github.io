App证书文件

发表于 2016-08-01   |   评论
什么是证书

证书分为两种，一种是开发证书Development，也就是开发过程中需要用到的证书，用于开发过程中的真机调试；Development;

第二种是产品证书Production，产品证书分为很多种：

Production Profile Name signing 作用
App Store and Ad Hoc Sign your iOS app for submission to App store for ad hoc distribution 这个是发布证书，用来发布到AppStore或者打ADHoc包的时候用到的证书
Apple Push Notification service SSL Establish connectivity between your notification server and Apple Push Notification service production environment.A separate for each app you distribute Apple Push Notification service SSL (Production)是推送证书，每一个应用需要一个独立的推送证书和苹果的推送服务器建立连接
Pass Type ID Certificate Sign and send updates to passed in Passbook Pass Type ID Certificate是做Passbook时需要申请的证书
Website Push ID Certificate sign and send updates for websites Website Push ID Certificate我也没用过，等以后用到在回来补充
VolP services certificate Establish connectivity between your server and the VolP service. A separate certificate is required for each VolP app you distribute. 暂无
Apple Pay Certificate Decrypt app transaction data sent by Apple to merchant/developer 配置Apple pay 的证书

over.

Identifiers标志符

在Identifier我们主要看的是App ID ,App ID 是一个用来标示我们应用程序App的，也就是我们的身份证一样，每一个应用程序都有一个唯一的App ID ,也就是我们工程中看到的Bundle ID 。App ID主要有一下两种：

App Id signing 作用
Explicit App Id if you plan to incorporate app services such as Game center, In-app purchase, Data protection, and iCloud, or want a provisioning profile unique to a single app ,you must register an explicit App ID for your app. To create an explicit App ID ,enter a unique string in the Bundle ID field. This string should match the Bundle ID of your app . 明确的App ID，这种标示符用于标示一个唯一的应用程序,他跟我们的Bundle ID一致，例如我们程序的Bundle ID为com.zhangmangyuan.ABC，那么我们的App ID也就是com.zhangmangyuan.ABC。
Wildcard App Id This allows you to use a single App ID to match multiple apps. To create a wildcard App Id, enter an asterisk(*) as the last digit in the Bundle ID field. 第一种用创建的App ID，他可以标示所有的应用程序；第二种是com.zhangmangyuan.创建的App ID，他可以标示以com.zhangmangyuan开头的所有应用程序.

Device 设备

这个最好理解，就是我们需要开发和调试的设备。

描述文件Provisioning Profiles

描述文件里包含了上述的三种信息：证书、App ID、设备Devices。描述文件也分为两种

4.1 Development的描述文件，也就是说在我们真机测试的时候选择了development描述文件，那么需要调试的工程必须要有响应的证书、App ID和将要运行到设备的标识

4.2 Distribution的描述文件，是发布证书的描述文件，原理和开发的描述文件类似。
