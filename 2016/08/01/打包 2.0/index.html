发表于 2016-08-01   |   评论

打包

打包是难免的，每次需要给测试、产品或者UI或者在没有加入调试的真机上看效果的时候需要打ipa包。打包方式有很多种，不乏黑客技。以下是app几种打包方式，部分还待整理，希望大家一起补充。

  • 脚本打包

    步骤：工程项目中有AutoArchive文件夹下的ipa_generator.sh脚本是用来打包的脚本工具，直接在终端运行该脚本即可。例如：

>

>

用脚本打包的速度貌似比Xcode打包快些，而且如果项目没有更改，再次打包的用时在1分钟左右。

注意：有时候用该脚本打出来的包版本不对，请先确定你需要打包的分支是dev还是其它版本。

>

  • Xcode打包

    原理：不详

    步骤：第一次用Xcode打包的孩子可能会遇到各种各样恶心的问题，我将我遇到的问题一一汇总

    1. 首先要选择真机，你的项目要在测试机上跑得起来。（跑不起来的话是证书的问题啦，这个自己先解决）

>

  1. 证书问题：比如 “ no provisioning profile …”, 仔细检查 是 TuNiuApp 还是App pro 或者 App WatchKit的证书问题导致的，去Build Settings—>Code signing identity中去配置吧。（此处不熬述）

  2. 遇到的第一个问题是

>

查看途牛项目发现有Crashlytics.framework，报错的原因是途牛项目中集成了这个崩溃统计框架，生成安装包需要导入Crashlytics才能继续，You must install Crashlytics to continue.这句话貌似就是解决方案咯。这是Crashlytics官网http://try.crashlytics.com/sdk/

下载这个软件安装一下就好了。不需要登录哦。然后就可以继续打包了。

  • jenkins打包

    当前面两种方式行不通的时候，才建议你用这种方式打包，因为jenkins打包速度实在太慢了，QAQ。还有一个比较恶心的问题是打包过程中，其它版本的已经构建好的包就会被覆盖掉，这就是传说中的单线程么？jenkins打包工具一般是测试人员使用的，少用这种工具，不要挤占测试人员的线程。

    具体步骤：

    1. 只有确保你需要打包的代码已经入库才能用jenkins打包，不然就悲剧了，白辛苦一番。

    2. 请先登录jenkins。然后找到这个东东

      http://172.30.38.117:8080/job/tuniu_app_inhouse/

    3. 选择左边栏的 “Build with Parameters”, 在右边栏 EMAIL_RECEVIER中输入你的email地址，如：XXXXXX@tuniu.com。 点击开始构建，在 左边栏的Build History 构建历史就可以看到你的＃xxx 项目正在构建中了，waiting吧。

    4. 构建完毕之后，选择该项目
